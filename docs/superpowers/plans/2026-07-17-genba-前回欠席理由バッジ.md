## 前提知識（この repo 固有・知らないと事故る）

1. **genba.html は CRLF 改行。** 検証は必ず node で実バイトを読む。**Read ツールの表示を根拠にしない**
   （本セッションで `コード.js` 12723行目が `\ ★…` と壊れて表示され、実体は `// ★…` だった実例あり）。
2. **本番HTMLを実ブラウザで開いて検証しない。** `file://` でも no-cors + keepalive で本番GASへ飛び、実データを壊す。
   検証は node / jsdom のみ。テストデータに**実在の利用者名を使わない**。
3. **テストは「実コード抽出」方式。** 別ファイルに写した複製ではなく、genba.html から出荷コードそのものを
   波括弧バランスで抽出して検証する（`scripts/test-genba-absence-slots.js` の流儀）。
4. **純関数は自己完結させる。** 抽出は `function NAME(...){...}` 単体しか拾わないため、
   関数外の定数を参照すると sandbox で `ReferenceError` になる。**マッピング表は関数内に置く。**
5. **本番反映は社長の手。** テスト緑 → 社長承認 → 版上げ → `git push` は社長 → `--verify` で証跡。
