## 1. 目的

やわらぎが取得している加算と事業所基本情報を、スタッフが1タップで見られる独立アプリ。
ケアマネ対応時にサービスコードを読み上げられるよう、各加算行にコードを併記する。

## 2. 調査で判明した前提の訂正（重要）

指示書の前提のうち、**2点が実態と食い違っていた**。設計はこの訂正を織り込む。

### 2.1 「共通トークン認証・アクセスログ」は存在しない作法

- master の `gas/yawaragi-board/コード.js` に `checkToken` / `recordAccess` の呼び出しは **0件**（実測）。
- `token-auth-core.js` / `access-log-core.js` はファイルとして置かれているが **git 未追跡**。
  `feat/gas-token-auth`（37コミット・origin/master に対し16299行削除の重度 stale）に取り残されている。
- 本番GAS @327 には配信済みだが `TOKEN_ENFORCE=OFF` 既定で何もしていない。
- トークンを送っている HTML アプリは **1本も無い**（`yukyu.html` の自前CFGトークンは無関係な別物）。

→ **社長判断: 版ゲートのみ（案A）**。既存57本と実際に同じ作法にする。
加算・事業所情報は重要事項説明書に載るレベルの情報で PII ではないため、
intake（利用者PII）のような fail-closed 認証は過剰。
将来 `feat/gas-token-auth` が master 反映されたら全アプリ横並びで有効化する。

### 2.2 単価 10.27円 はコードベースから裏取りできない

- リポジトリ全体に `10.27` は不在。
- 逆に伝達ボード seed に未完タスクとして残存:
  `コード.js:15355` … 「🔴 東松山市の総合事業/介護予防の正確な単価を調べる｜6級地。出所＝市の総合事業の手引き／運営推進会議資料」
- 計算上は整合する（6級地の上乗せ 6% × 通所介護の人件費割合 45% → 10円 × 1.027 = 10.27円）が、これは一次情報ではない。

→ **社長判断: 10.27円を確定値として表示（介護給付のみ）**。総合事業3保険者分は「未取得」と明示。

### 2.3 指示書の穴: 総合事業の保険者3つ分の級地・単価が空白

指示書は保険者を「介護給付＝東松山市のみ／総合事業＝東松山市・嵐山町・川島町」とするが、
地域区分は「東松山市＝6級地」しか無い。総合事業の単価は市町村が定めるため、
**嵐山町・川島町の級地／単価は現時点で未取得**。上記2.2の未完タスクが総合事業側を名指ししているのと符合する。

→ v1 は「総合事業の単価は未取得」と画面に明示する。判明したらシートに打つだけで出る。

### 5.2 ★罠対策: コード列のテキスト固定

Sheets は `781241` を**数値化**する（`A61111` は英字混在なので文字列のまま）。

- `setNumberFormat('@')` でコード列をテキスト固定（伝達ボードの期限列と同じ手当て）
- 純関数側でも `String()` 正規化して二重に守る

## 6. 純関数（TDD先行）

`gas/yawaragi-board/kasan-core.js` / テスト `scripts/test-kasan-core.js`（既存命名 `test-*.js`・node直実行）。

指示書「表示整形・並び順のピュア関数を先にテストする」に対応。

| 関数 | 責務 |
|---|---|
| `kasanParseRows(values)` | シート2次元配列 → オブジェクト配列。**ヘッダ名で列解決**（列追加に強い） |
| `kasanNormalizeCode(v)` | コード正規化（数値化された `781241` を文字列へ復元・`A61111` は素通し） |
| `kasanSortRows(rows)` | 表示順 → コードの安定ソート |
| `kasanGroupBySection(rows)` | section 別に束ねる |
| `kasanSplitKeitou(rows)` | 加算を `介護給付` / `総合事業` の2系統に分割 |

### 6.1 テスト観点

- `kasanNormalizeCode`: 数値 `781241` → `'781241'`／文字列 `'A61111'` → `'A61111'`／空 → `''`
- `kasanParseRows`: 列順を入れ替えても正しく解決／ヘッダ欠落時に落ちない
- `kasanSortRows`: 表示順が同値ならコード順で安定／表示順が空なら末尾
- `kasanGroupBySection`: 未知 section を捨てない（黙って消さない）
- `kasanSplitKeitou`: 系統が空の加算行を落とさず「系統不明」として拾う（沈黙させない）
