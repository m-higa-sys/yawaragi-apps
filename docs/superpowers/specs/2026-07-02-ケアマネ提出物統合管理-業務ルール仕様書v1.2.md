## 0. スコープ

- 目的: **月末にケアマネへ渡す書類**の工程管理（作成〜送付）＋その前提となる内部工程（測定・モニタリング・口腔体操実施等）の実施管理
- 最重要要件: **繰越管理**。今月出せなかったタスクは「何月分」の月ラベルを保ったまま、送付済になるまで翌月以降の画面に残り続ける
  - 遅延は常態として設計する（ケアプラン作成の遅れ、利用者欠席でサイン取れず、作成忘れ等）。月末一括渡しは目標であって前提ではない
- 対象外: LIFE提出、Google Drive保存運用、利用者へ渡す控えの管理
- 対象に含む: 口腔体操の**実施記録**（ケアマネ提出物ではないが運営指導時に提示が必要）

## 1. 基本原則

1. **ケアプランが土台（主従原則）**: 全ての加算計画書はケアプランに従属する
   - ケアプランが変わった時点で、加算を取っている**計画書すべてがリセット**される
   - 各計画書の期間はケアプラン期間の範囲内で設定（例: ケアプラン1ヶ月なら全計画書も1ヶ月）
   - ケアプラン1年 → 個訓・口腔は3ヶ月ごと、通所介護計画書は1年
   - ケアプラン2年 → 通所介護計画書は**上限1年**で設定
2. **結果に対して計画書を作る**（全計画書共通）。測定・評価が先、計画書が後
3. **前月準備の原則**: N月開始の計画書は、N−1月中に「測定（＝結果報告書）→ 計画書作成 → 利用者署名」まで完了させ、N−1月末にケアマネへ渡す
4. **対象月キー＝作業月**: 全タスクは「◯月分の◯◯」として**作業すべき月**を明記して管理する（対象月台帳 v2設計を採用）。1月開始の計画書は「12月分: 個訓計画書（1月〜）」
   - **サイン期限ルール（加算要件）**: 計画書は適用開始月の**本人初回利用日まで**にサイン・同意が取れないと加算が取れない。繰越してもこの期限は消えない
   - 期限の自動計算: 契約曜日・出欠データから翌月初回利用日を算出し、期限バッジ表示（例:「⚠1/6初回利用までにサイン必須」）
5. **繰越ルール**: 計画書・モニタリング・評価・測定結果は翌月繰越可。**実績のみ繰越不可**（当月末厳守）
   - 月替わり時に未送付タスクは自動繰越（操作不要）
   - 繰越タスクには**理由チップ**をタップで付与: 🖊サイン待ち（利用者欠席）／📋ケアプラン待ち／✏️作成遅れ／🏢ケアマネ都合／その他
   - 理由は必須にしない（作業をブロックしない）が、未設定は「❓理由」バッジで管理者ビューに目立たせる
   - タスク表示に理由バッジを常時併記（例:「12月分: 個訓計画書（1月〜）🖊サイン待ち」）
   - 管理者ビューに理由別集計を表示（作成遅れの偏り＝属人化対策、ケアプラン待ちの偏り＝事業所への働きかけ、の判断材料）
6. 周期は**ローリング計算**: 次回期日 ＝ 前回作成日 ＋ 基本周期（カレンダー固定ではない）

## 2. 利用者区分と書類セット

利用者の区分（要介護／要支援・事業対象者）で表示する書類群を切り替える。

### 2-1. 要介護

| 書類 | 周期 | 種別 | 備考 |
|---|---|---|---|
| 個訓 結果報告書＋計画書 | 基本3ヶ月 | 送付 | **2個1セット送付**。測定＝評価＝結果報告書。同じ作業月内で報告書→計画書→署名 |
| 口腔 計画書＋評価 | 基本3ヶ月 | 送付 | 評価と計画書は同月。口腔フラグONの利用者のみ |
| 口腔 モニタリング | 毎月 | 内部 | 実施のみで完了（提出なし） |
| 口腔体操 実施記録 | **月2回** | 内部 | 既存口腔アプリで稼働中。運営指導対応 |
| 通所介護計画書 | 満了月（基本12ヶ月、半年もあり、最長1年） | 送付 | 満了日ベース。満了月に計画書作成（モニ・評価工程なし） |
| 実績 | 毎月 | 送付 | 事業所単位。繰越不可 |

### 2-2. 要支援・事業対象者

| 書類 | 周期 | 種別 | 備考 |
|---|---|---|---|
| 測定結果 | **4ヶ月固定**（変動なし） | 送付 | 呼称は「測定結果」で統一。義務ではないがケアマネへ測定結果報告として提出。誰が測定するかを一目で分かる表示 |
| 口腔 計画書＋評価 | 基本3ヶ月 | 送付 | 要支援も口腔対象（フラグONの利用者） |
| 口腔体操 実施記録 | **月1回** | 内部 | 既存口腔アプリで稼働中 |
| 通所モニタリング | 毎月（計画期間の最終月を除く） | 送付 | 要介護には存在しない |
| 通所評価＋通所計画書 | 計画期間の**最終月** | 送付 | 独立した2チェック（束ねない） |
| 実績 | 毎月 | 送付 | 同上 |

### 2-3. 通所介護計画書の工程構造（要支援・事業対象者）

計画期間がNヶ月の場合:
```
1ヶ月目 〜 N−1ヶ月目: 毎月モニタリング
最終月（Nヶ月目）    : モニ抑制 → 評価 → 計画書作成（次期間の計画書）
```
例: 12ヶ月計画なら1〜11ヶ月目がモニタリング期間、12ヶ月目に評価＋計画書。

### 2-4. 口腔対象フラグ

- 原則全利用者（要介護・要支援とも）が口腔対象（約95%）。非該当・非同意の約4〜5名を除外
- 新規時のチェックフロー: 加算該当チェック → 該当なら加算取得 → 同意確認 → 同意者のみ口腔タスク生成
- 利用者マスタに口腔対象フラグを持つ

## 3. 周期・イレギュラールール

- 基本周期: 個訓3ヶ月／口腔3ヶ月／測定結果4ヶ月（固定）／通所はケアプラン期間（上限1年）
- **短縮の唯一の原因はケアプラン**: ケアプラン変更（区分変更・中途終了を含む）により2ヶ月・1ヶ月になる
- 優先順位（確定済み3段階）: **手動上書き ＞ 実満了日（実データ） ＞ 周期計算値**
- 満了日未設定者は⚠バッジ表示（無言で落とさない）

## 4. 新規利用者の初回フロー

- 通所介護計画書: **契約終了後から作成可能**（初回利用前に作成できる唯一の計画書）
- 個訓・口腔: **初回利用当日に完結**——当日に機能評価（測定）＋口腔測定 → その日に計画書作成 → その日にサインまで取る
- 原則、全員フルセットで開始（口腔非同意者を除く）
- 入所アプリ（P5）登録 → 初回タスク自動生成の連携を行う

## 5. ステータス変化イベント

| イベント | タスクへの影響 |
|---|---|
| **ケアプラン変更** | **全計画書リセット**（新ケアプラン期間を起点に各周期を再設定） |
| 区分変更確定 | ケアプラン変更の一種。**区分変更アプリ（P4）と連携**。書類セット切替＋周期リセット |
| 区分変更中 | 実績のみ送付不可（Phase 3実装済みルール）。他書類は可 |
| 利用中止 | 即終了。未完了タスクは打ち切り消込（理由を記録） |
| 長期入院 | 計画書を一旦切る場合あり → タスクを保留（休止）。利用再開時に初回セットを再生成 |
| 月内利用ゼロ | 当月実績の対象外 |

## 6. 送付・期限ルール

- **月末一括渡し（目標）**: 当月実績＋翌月開始の計画書類を当月末にケアマネへ。計画書類は遅延時に繰越、実績は繰越不可
- 実績: ケアマネの国保連提出（毎月1〜10日）を死守
- **送付方法4パターンと出力形態**:

| 送付方法 | 実績 | 計画書類 | 備考 |
|---|---|---|---|
| ケアプー | データ（CSV）送信 | **PDF化して添付** | 提供票の受領が前提。未受領は従来方式に切替。徐々に全事業所移行見込み |
| メール | PDF | PDF | |
| FAX | PDF（秒速FAX） | PDF（秒速FAX） | |
| 持参 | 印刷 | 印刷 | |

- **署名フロー**: サインが必要な計画書は 印刷 → 利用者署名 → （メール/FAX/ケアプーなら）PDF化、（持参なら）そのまま印刷物を持参
- 送付方法は事業所単位の既定値＋利用者単位の上書き可
- 各タスクに送付方法由来の**出力形態ガイド（PDF化／印刷）を自動表示**（個訓評価の出し分け構想を全書類に拡張）

## 7. 担当・作業記録

- 担当方式: **自由取り＋実施者・日時の自動記録**（updatedBy / updatedAt 全操作必須）で開始。1〜2ヶ月計測後に割り当て制を判断
- 測定の担当者が一目で分かる表示（要支援・事業対象者の測定を含む）
- 月次で担当者別の完了件数を集計し、管理者ビューに表示（属人化の見える化）

## 8. UI安全ルール（全ボタン共通）

1. 二重タップ防止: タップ即ボタンロック＋「保存中…」表示。バックエンドは冪等upsert
2. 破壊的操作は確認ダイアログ必須。対象を具体表示（例:「16名を送付済みにします」）
3. チェックはトグル式（タップで解除可）。操作直後に「元に戻す」5秒表示
4. 楽観更新（押した瞬間に画面反映）→ 裏でGAS保存 → 書き込み検証ピル表示（genba.htmlパターン）
5. 同期: 真実はGoogleシートのみ。localStorageはオフラインキャッシュに格下げ。開いた時＋タブ復帰時＋定期ポーリングで再読込、「最終同期時刻」を常時表示

## 9. 用語の統一

- 「測定結果」＝ 要支援・事業対象者の4ヶ月測定（「アウトカム」は使わない）
- 個訓では 測定 ＝ 評価 ＝ 結果報告書（同一のもの）
- 「口腔」（音声入力の「航空」は口腔のこと）

## 10. 要確認ポイント → **全解決済み**

- 対象月ラベル → **作業月キーで確定**（理由: 加算要件上、適用開始月の初回利用日までにサインが必要なため、作業月＝12月分として管理しないと期限が守れない）
- 実施記録の粒度 → 要介護 月2回・要支援 月1回の実施回数管理（既存口腔アプリで稼働中）
- 要支援の口腔 → 対象に含む（§2-2に追加済み）
