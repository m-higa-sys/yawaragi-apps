# ケアマネ送付チェックリスト：送付方法別グループ表示 設計

- 日付: 2026-06-27
- 対象: `ケアマネ送付チェックリスト.html`（GitHub Pages公開・no-storeアプリ）
- 区分: フロント並べ替えのみ（GAS無改修・版上げ不要）

## 目的

ケアマネ送付の「送る作業」を送付方法ごとにまとめて実施したい（ケアプーまとめてアップ／FAXまとめて流す等）。
現状の「事業所別」「利用者別」に加え、**送付方法でクラスタした第3のビュー**を追加する。

## 要件

1. 「表示」プルダウンに `送付方法別` を追加（`事業所別`/`利用者別` に続く3つ目）。
2. グループ順（上→下）: **持参 → ケアプー → メール → FAX → その他**。
3. グループ内の事業所並びは現状「事業所別」のロジックをそのまま維持（人数多い順＋同一敷地ペア隣接）。
4. 各グループ先頭にグループヘッダー（アイコン＋名称＋件数）。
5. 送付方法の判定は「送付用居宅一覧」の送付方法列（`contact.method`）を正本に使う。旧ハードコードmapは使わない。
6. 「その他」は4分類のどれにも一致しない事業所（郵送・未登録・想定外文字列）の受け皿。**0件のときだけ非表示**。

## 触らない / 制約

- `office`/`user` ビューのコードパスは現状維持。`method` 分岐を足すだけ。
- 完了事業所の下部集約（送付完了セクション）は `office`/`user` ビューのまま据え置き。
- localStorage/オリジン分裂には触れない。版上げ不要・GAS無改修。

## 設計

### データ源（正本確認済み）

`allData.contacts ← YAWARAGI_BOARD_API?action=contacts → cmContacts`（送付用居宅一覧）。
`contactMap[office].method` が送付方法の正本。送付方法のハードコードmapは存在しない。

### グルーピング

- `viewMode === 'method'` は `user` 以外なので、既存の「事業所キーで group 化＋人数多い順＋同一敷地ペア隣接」の `sortedOffices` 算出をそのまま再利用する（実質 `office` ビューと同じ並び）。
- 各事業所の方法グループは下記ヘルパーで判定:

```
methodGroupOf(office):
  contact = contactMap[office]
  if resolveSoufu(contact.method).isCarePlan -> 'careplan'
  m = contact.method
  if m includes '持参' -> 'visit'
  if m includes 'メール' -> 'mail'
  if m includes 'FAX'  -> 'fax'
  else -> 'other'
```

- グループ定義（順序＝表示順）:

```
METHOD_GROUPS = [
  visit    🚶 持参
  careplan 📡 ケアプー
  mail     📧 メール
  fax      📠 FAX
  other    📦 その他
]
```

### 描画

- `officeBlocks`（filteredOffices 順を保持）に各ブロックの所属グループ `group` を持たせる。
- `method` ビューのみ、incomplete/complete 分離を使わず、`METHOD_GROUPS` 順にループ。
  - そのグループのブロックが0件ならスキップ（＝空グループ・空「その他」は非表示）。
  - 非空グループはヘッダー（`アイコン 名称 (n事業所)`）＋グループ内ブロックを出力。
  - グループ内順序は filteredOffices 順＝事業所別と同一（持参ペア隣接も自然に維持）。
- `office`/`user` ビューは従来の incomplete/complete 分離出力のまま。

### 完了済みの扱い（確定）

送付方法別ビューでは完了事業所も各方法グループ内に残す（下部集約しない）。
理由: 一括送付作業中はそのグループの全体像（残り/完了）が見える方が「全部流したか」確認に向く。
完了はカード右の `100% (n/n)` で判別可能。

## 完了条件

- [ ] method判定が `contact.method`（送付用居宅一覧）を正本に見ている（旧ハードコードmap不使用）。
- [ ] 4分類に該当しない事業所が必ず「その他」に出る（消えない）。
- [ ] グループ順が 持参→ケアプー→メール→FAX→その他。
- [ ] 「その他」が0件のときだけ非表示。
- [ ] 「事業所別」「利用者別」は従来通り（並び・完了分離とも不変）。
- [ ] 実装後、「その他」に何件入ったかを社長へ報告（郵送が居れば独立グループ化を再検討）。
