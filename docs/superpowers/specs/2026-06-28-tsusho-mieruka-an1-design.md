# 通所計画書 見える化（案1・clasp不要）表示仕様

- 作成: 2026-06-28
- 対象: `monitoring.html` 計画書タブ（#panel-plans）のみ・**全108名（要介護55＋要支援53）**
- 制約: **frontendのみ・GAS書き込みなし・clasp不要**。データモデルは変えない（既存 `最終作成日(plan_date)`＋`サイクル(cycleMonths・現状全員12)` を読むだけ）。
- 位置づけ: design2「満了日期限管理」の**前哨（暫定見える化）**。正確な満了日（min(ケアプラン,1年)）は次回の案2（GAS改修・clasp要）。

## 背景（実測で判明した現状）

- 計画書タブは要介護も要支援も**全108名**が並ぶ（[getTsushoPlansYear] users＝全員・filterはcategory非依存）。
- `cycleMonths` は **108名全員12固定**（個別設定された人ゼロ）。records（最終作成日）は**ほぼ空**＝大半が未入力。
- 未入力カードは現状 **ピンク背景(cell-todo)＋⚪絵文字** のちぐはぐ。「未設定＝危険」と読めず、`⚪未入力`フィルタも中立な名前で埋もれる。

## 主役：未入力者の可視化（穴#1）

1. **状態を4→5分類に。未入力を明示の“危険”に格上げ**：
   - **未設定**（最終作成日なし）＝ **⚠ 赤バッジ「最終作成日 未設定」**。絵文字 ⚪ → **⚠**。カードは既存の薄赤(cell-todo)を活かしつつ、バッジで「未設定＝要対応」と明言。
   - 🔴 期限超（経過≧サイクル）／🟡 もうすぐ（≧5/6）／🟢 OK は据え置き。
   - ＝ 未設定(⚠) と 期限超(🔴) を**別物として区別**（両方“赤い”が意味が違う）。
2. **進捗カウンタ**（スタッフが少しずつ埋められるよう）：
   - 計画書タブ上部に **「📊 最終作成日 未入力 N / 108人」** を常設表示。Nが減るのが見える。
3. **フィルタ名の整合**：状態フィルタの `⚪未入力` ボタン → **`⚠未設定`** に改名（data-status="none" は不変）。押せば未設定者だけに絞れる＝上から埋められる。

## 補助：満了予定日＋残日数（暫定・サイクル依存）

4. **満了予定の表示**（最終作成日がある人のみ）：
   - 計算＝ `最終作成日 + サイクル(月)`。例: 2025-06-30 + 12ヶ月 → 満了予定 2026-06-30。
   - 表示＝ **「≈満了 6/30 ・ あと12日」**（期限内）／ **「満了 6/30 ・ 15日超過」**（期限超・赤）。
   - **暫定であることを必ず明示**（半年の人のズレを鵜呑みにさせない）：
     - 行頭に **`≈`**（近似マーク）。
     - 文言に小さく **`(暫定・サイクル12ヶ月)`** を併記。
     - タブ上部に注記1行：**「満了予定はサイクル（現在全員12ヶ月）からの暫定計算。正確な満了日は次回対応。」**
   - 未設定の人には満了行を出さず、代わりに ⚠未設定 バッジのみ。

## 触らないもの（スコープ外）

- モニタリング暦タブ（要支援送付管理）。
- サイクル/最終作成日/送付チェックの**入力・保存ロジック**（既存のまま）。
- GAS・データモデル・満了日の正確化（案2）。

## 実装メモ（monitoring.html）

- `computeMantan(lastDate, cycleMonths)` を追加（満了予定日・残日数を返す。lastDate無→null）。
- `renderTsushoPlans()`（[L1464-1542](../../../monitoring.html#L1464)）のカード組み立てで：
  - 未設定判定→⚠バッジ＋絵文字差し替え。
  - 満了予定行を追加（暫定マーク付き）。
  - cards の前に **未入力進捗バー**＋**暫定注記**を prepend。
- 状態フィルタボタン（[L575](../../../monitoring.html#L575)）のラベルを `⚠未設定` に。
- 全て読み取り表示のみ＝**保存系・GAS呼び出しは一切追加しない**。

## 確認観点（社長の実機）

1. 未入力の人が **⚠未設定（赤）** で目立つ（⚪で埋もれない）。
2. 上部に **「未入力 N / 108人」** が出て、入力するとNが減る。
3. 最終作成日がある人に **「≈満了 M/D・あとN日」** が出る。
4. 満了予定に **暫定（サイクル依存）と分かる印**がある。
5. 要介護も要支援も同じく出る（全108名対象）。
6. 入力・送付チェック・モニタリング暦は従来通り動く。
