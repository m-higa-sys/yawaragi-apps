- **日付**: 2026-07-11
- **対象アプリ**: genba.html 内「伝達ボード」（実体は genba.html で確定・データ源＝利用者台帳「伝達ボード」タブ）
- **正本GAS**: my-project（clasp管理・追記のみ・165関数は無傷）。このrepoの `gas_yawaragiボード.gs` は古いミラーで触っても本番反映されない
- **ステータス**: 設計確定 → writing-plans へ

---

## 0. 背景と目的

全員宛て投稿で「見てない人がいるかも」と思うと完了を押せない／いつまでも残る。
既読を可視化し、**未読者の名前を主役に表示**することで「見てって言えばいい」の運用を成立させる。
全員既読になったら完了ボタンが変化し、安心して押せるようにする。**自動完了はしない**（終わるまで方式を維持）。

### 不変条件（この設計の背骨・実装者が同意の上で崩さないこと）

過去に踏んだ地雷（save_haichi のクライアント全上書き＝同時編集を取りこぼす誤発火）を二度踏まないため、以下を厳守する。

- **不変条件1**：`recipients` は **add_dengon_message でサーバ側でマスタを引いて確定**する。クライアントは recipients を計算・送信しない。フロントの人数表示は「表示用の一次確認」であって保存値の起源ではない。
- **不変条件2**：`isAllRead` は **recipients と readBy の2つだけで判定**する（マスタの在籍状態を二次参照しない＝退職者は除いて判定…等の親切を足さない。動く分母の誤発火を防ぐ）。
- **不変条件3**：`mark_dengon_read` は、押された名前が **その投稿の recipients に含まれるかをサーバ側で検証**し、含まれなければ却下する。

- 比嘉は 代表 / − で登録 → 条件式だけで「職種・雇用区分宛てには入らない（全員には入る）」が自動成立（ハードコード不要）。

### A-2.「伝達ボード」タブに2列追加（末尾）

既存列：`id | from | to | body | deadline | createdAt | 完了フラグ | 完了日時 | doneBy`

- `recipients`：投稿時に確定した対象者名の JSON 配列（**スナップショット**）。個人宛て・社長宛ては空。
- `readBy`：既読者名の JSON 配列。

既存投稿は両列が空 → エラーにせず「全員未読」扱いで表示（互換）。

## B. GAS（my-project 正本・追記のみ）

| action | 種別 | 役割 |
|---|---|---|
| `dengon_staff_master` | GET/JSONP | 「スタッフ」シートを返す `{staff:[{name,role,employ,active}]}` |
| `add_dengon_message`（改修） | POST | グループ宛てはサーバ側でマスタから対象者算出→ `recipients` 保存（不変条件1） |
| `mark_dengon_read` | GET/JSONP | サーバ側 read-modify-write で readBy に1名追加（冪等・read-back検証・recipients照合＝不変条件3） |
| `unmark_dengon_read` | GET/JSONP | readBy から1名除去（冪等） |
| `dengonBoard` / `dengonHistory`（改修） | GET | 返却に `recipients` / `readBy` を追加 |

### 宛先算出（マスタ動的・サーバ側）

- 全員 = active 全員（11）
- 全員・ドライバー除く = active ∧ 職種≠ドライバー（9）
- 社員 = active ∧ 雇用区分=社員（2）
- 相談員 = active ∧ 職種=相談員（3）
- 看護師 = active ∧ 職種=看護師（3）
- 個人 = active 全員（比嘉含む）

### readBy 更新の掟

- クライアントが配列を丸ごと送るのは**禁止**（save_haichi 上書き事故の再発防止）。
- 必ずサーバ側で該当行だけ **read-modify-write**（予約memory「mark_read 先行」を厳守）。
- `mark_dengon_read` は書き込み前に **recipients 照合**（不変条件3）。対象外の名前は却下。

## C. フロント（genba.html）

### 宛先 select 刷新

全員(11) / 全員・ドライバー除く(9) / 社員(2) / 相談員(3) / 看護師(3) / 特定スタッフ。
人数はマスタから算出表示（＝分母の一次確認を兼ねる。※保存値の起源ではない・不変条件1）。
特定スタッフ（個人）はマスタ active 全員（比嘉含む）。

### 既読UI（グループ宛てのみ・個人宛てには出さない）

- カード下に `recipients` の名前チップを表示。
- 未読は赤系で「未読: 工藤・大久保」と**名前を主役**に。既読チップは薄く。
- チップ tap →「◯◯さんとして既読にしますか?」→ `mark_dengon_read`。
- 既読チップ再 tap →「既読を取り消しますか?」→ `unmark_dengon_read`。
- **楽観更新しない**。サーバ確定後に再描画。

### 完了ボタン変化

- `recipients` 全員が `readBy` に含まれたら → 「✅ 全員既読・完了にする」＋色変化。
- 未読が残っても従来通り押せる（お願い型タスクのため）。
- 完了の挙動自体は不変（既存2段階確認・`completeDengonMessage` はそのまま）。

### 身元

- チップ tap ＝その人が押す（施設共有スマホのオナーシステム）。投稿者選択 `dengonSelectedStaff` とは独立。

### 個人宛て・社長宛ての扱い

- recipients が空（個人宛て・社長宛て）は既読UIを出さない。従来通りの完了ボタン。
- 既存グループ投稿（recipients 無し・移行前）は表示時に現行マスタから算出フォールバックし「全員未読」で描画（新規は必ず recipients が入るので、この経路は移行期のみ）。

## D. テスト（TDD・純関数抽出）

指示書の検証5項目に対応する純関数を genba.html から抽出してテスト（既存 `scripts/test-genba-*.js` の extractFn 流儀に合わせる）。

- `computeRecipients(master, group)` → 各分母が 11/9/2/3/3、比嘉の包含（全員系）/除外（職種・雇用区分系）
- `addReadBy` / `removeReadBy`（冪等・二重追加/存在しない除去で壊れない）
- `isAllRead(recipients, readBy)` → **recipients と readBy のみで判定**（マスタ非参照・不変条件2）
- 既存投稿（両列空）の非破壊表示
- GAS側は clasp窓で擬似実行 → シートに readBy が入る/消えるを**一次情報で実測**
