# 【別トラック要件メモ】欠席box 早すぎる送信の抑止（時刻警告ポップアップ）

- 日付: 2026-07-07
- ステータス: **要件メモのみ（今回の-25には含めない）**。box本体push後に別polishとして実装。
- 経緯: box改修(移設+polish)の-25着地を優先するため、送信ボタン挙動への変更（これまでの「表示のみ非接触」から踏み込む）は今回スコープ外とした（クロ・社長判断 2026-07-07）。

## 要件
送信押下時、早い時間帯（**閾値は社長確定**・例: 午前9時前 / 午後13時前）は、最終サマリーに
警告を出して二択確認する:

> 「利用開始前です。キャンセル時に取消連絡が必要になります」
> [送る] / [やめる]

- 通常時間帯（閾値以降）は既存の最終サマリーのみ（挙動不変）。
- 目的: **利用開始前送信による取消メール問題の予防**（早朝に送ってしまい、その後キャンセルになると取消連絡が発生する）。

## 実装上の注意（着手時）
- 変更対象は送信フロー（kbOpenSummary / kbExecuteSend 周辺）＝今回の「表示のみ非接触」原則を超える。
- 当日ガード（kbIsViewToday_）・originガード（gnbGuardProdWrite）・二重送信ガード（kbCollectSendTargets_/kbIsAlreadyNotified_）は**非接触維持**。警告は「送信の一拍手前」に挟むのみで、送信対象集合やガードは変えない。
- 閾値時刻は社長確定が前提（未確定のまま実装しない）。
- 自動送信は復活させない（手動送信のまま）。本メモは「手動送信の押下タイミングを早すぎる時に警告する」だけ。
- TDD: 「閾値前=警告分岐あり／閾値後=既存サマリーのみ」をR群相当の構造証明＋純関数（時刻→警告要否）で固める。
