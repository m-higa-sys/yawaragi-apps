## 2. データモデル（シート設計・全て純粋追加）

### A. 提出送付台帳（新シート・システムの背骨）

| 列 | 型 | 説明 |

| status | enum | `未作成` / `揃った` / `送付済`（v1.2は基本踏襲・背骨§4） |
| sorotta_at / sofu_at | datetime | 状態遷移時刻 |
| soufusha | string | 送付済を押した人（将来のスタッフ委譲の土台） |
| soufuHouhou | enum | 持参/ケアプー/メール/FAX（事業所既定値＋利用者上書き。`2026-06-27-soufu-houhoubetsu-group-design.md` 接続） |
| kurikoshiRiyu | enum | `サイン待ち`/`ケアプラン待ち`/`作成遅れ`/`ケアマネ都合`/`その他`/空（空=❓バッジ。v1.2 §1-5） |

- **凍結レコードと生再計算の分離**（背骨§3補正3）: 台帳に書くのは「揃った/送付済/繰越理由」等の**確定した状態だけ**。「未作成」は行を書かず毎回算出（後から planStart 等が変わっても確定済みレコードが壊れない）。
- `jisseki` は userId 単位で行を持ち、**表示は事業所集計**（現行localStorageも利用者単位・Phase3ルール「区変中は実績のみ送付不可」と整合）。繰越不可＝taishoTsuki が過去でも翌月リストに出さず「未送付のまま月超過」を管理者ビューで警告。

### B. 対象月変換層（シートではなくロジック・背骨§3を v1.2 で拡張）

| docType | 対象月（作業月）の立て方 |
|---|---|
| tsusho_keikaku / tsusho_hyouka | 満了月の**前月**を作業月に（v1.2 §1-3 前月準備の原則）。満了月= 3段階優先「手動override ＞ 実満了日(`getTsushoDueDates`) ＞ planStart+11計算」。満了日ベースは**完了まで永続**（地平で切らない） |
| tsusho_moni | 当月（要支援・計画期間の最終月を除く） |
| kokun_set | `isPlanMonth`（shared.js:405）の真の月の**前月**。※v1.2 の2個1セット化により、`isHyoukaMonth` は kokun_set 内の工程（測定→報告書→計画書→署名）に吸収 |

| oral_moni | 当月（毎月・内部） |

| jisseki | 当月 |

- サイクル系の遡及地平は**直近3ヶ月**から開始（運用で調整可・背骨§3）。
- 「前月準備の原則」の適用: v1.2 §1-3が正。既存 shared.js の判定（当月主義）は**触らず**、変換層で前月へシフトする。ケアプラン台帳（C）導入後は周期起点をケアプラン期間から導出する判定に**新関数で**置き換え（既存 isPlanMonth 等は不変更）。

### C. ケアプラン台帳（新シート・主従原則の正本）

| 列 | 型 | 説明 |
|---|---|---|
| userId | string | |
| carePlanStart | YYYY-MM-DD | ケアプラン開始日 |
| carePlanMonths | int | 期間（月数。2年は通所上限1年で丸めず生値保存・導出側で上限適用） |

## 4. 画面設計（teishutsu.html 新設・no-storeアプリ＝版ゲート不要）

### 4-1. スタッフビュー（既定）

- **「今月送るリスト」** = `対象月 ≤ 今月 ∧ status ≠ 送付済`。事業所グループ表示（送付方法順: 持参→ケアプー→メール→FAX）。
- 行表示: `12月分: 個訓計画書（1月〜）🖊サイン待ち ⚠1/6初回利用までにサイン必須`
  - 月ラベル=作業月キー（taishoTsuki）＋適用月（tekiyoTsuki）
  - 繰越理由チップ（タップで付与・任意・v1.2 §1-5）
  - サイン期限バッジ（§4-3）
  - 出力形態ガイド（送付方法由来: PDF化/印刷・v1.2 §6）
- 操作: **「揃った」1タップ**（各書類担当）／「送付済」1タップ（当面社長・soufusha記録）。
- 区分（要介護/要支援・事業対象者）で書類セット出し分け（v1.2 §2）。
- 件数: 「今月あと○件」= 個訓のcount母型（個訓.html）流用・台帳status集計。

### 4-2. 管理者ビュー

- 繰越理由の別集計（作成遅れの偏り=属人化／ケアプラン待ちの偏り=事業所への働きかけ）＋❓理由未設定一覧。
- 担当者別 完了件数（updatedBy月次集計・v1.2 §7）。
- 実績の月超過警告（繰越不可のため別枠）。
- 満了日未設定者⚠一覧（無言で落とさない・v1.2 §3）。

### 4-3. サイン期限の算出

- 期限 = 適用開始月（tekiyoTsuki）の**本人初回利用日**。台帳「利用曜日」から適用月の最初の該当曜日を求め、欠席登録（板GASの欠席データ）があれば次candidate へ送る。結果を台帳 `signKigen` にキャッシュ（表示都度再計算でもよい・実装時に決定）。
- 未確定要素（祝日・臨時休業）は「⚠目安」表示で正直に出す。

### 4-4. UI安全ルール（v1.2 §8 全適用）

二重タップ防止＋「保存中…」／破壊的操作は対象具体表示の確認ダイアログ／トグル式＋「元に戻す」5秒／**楽観更新→GAS保存→別GET読み直しの書き込み検証ピル**（genba④方式）／真実はシート・localStorageはオフラインキャッシュ格下げ・「最終同期時刻」常時表示。

### 4-5. 既存localStorage進捗の移行

初回ロード時に `yawaragi_cm_send_progress_v1` を検出→「旧チェックをクラウド台帳へ取り込みますか？（○件）」→ upsert一括→ 取込済みマークを付けて以後は台帳が正。旧送付チェックリストは読み取り専用の参照期間を経て引退。
