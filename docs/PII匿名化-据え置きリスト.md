# PII匿名化 据え置きリスト（触ると本番が壊れる機能データ）

**目的**: リポジトリ内のPII（実在の利用者名・ケアマネ名・事業所名）を匿名化する際、
**匿名化してはいけない「機能データ」**を明示する。ここに挙げた定数・リテラルは、
実在の名称だが「表示・照合・送付判定に使う稼働ロジックの一部」であり、
匿名化するとアプリの動作が壊れる。将来の匿名化パスは**必ず除外**すること。

> このメモには実名を書かない。定数名と行番号（±ずれ得るので定数名を正とする）で示す。

## 匿名化してよい（機能に影響しない）
- ソースコメント（`//` `/* */` 行）内の例示氏名
- テストデータ（`scripts/test-*.js` / `scripts/verify-*.js` / 直下 `test-*.js`）の氏名・カナ
- 設計/引き継ぎドキュメント（`docs/**` の prose）・モックHTMLの表示例

## 宿題（設計課題）
`SHIEN_SOKUTEI_PAPER_SEED`（60名）は本番動作に実名が必要なため、匿名化で対応できない。
**恒久策はコードから実名データを剥がすこと**＝シード名を非公開データソース
（スプレッドシート等）から実行時に読む設計へ移し、リポジトリ（公開）には実名を置かない形にする。
それまでは本ファイルを匿名化対象から除外し続ける。

## 網羅性の担保方法
実名の「消し漏れ」判定は、利用者台帳（**終了者・中止者を含む全件**）を正本にする。
`riyousha-daichou-api` を `?includeEnded=1` で読むと全氏名が得られる。
取得した氏名リストでリポジトリ全体を再スキャンし、上表の機能データ以外のヒットを
コメント/テストデータ/ドキュメントに分類して潰す。
※ 取得した氏名リストはリポジトリ外（コミットされない場所）に置くこと。
